# 통합 계약 (Integration Contract)

> 이 문서는 **팀원이 합의 없이는 서로 못 붙는 접점만** 정의합니다.
> 프레임워크·라이브러리·폴더 구조·스케줄러·캐시 구현·AI 제공자·배포 타깃은 **팀 자유**입니다.
> 제품 기획은 [`../hddb_guidelines.md`](../hddb_guidelines.md) 참고.

## 0. 범위

| 합의 필요 (이 문서)                   | 팀 자유 (코드에서 결정)           |
| ------------------------------------- | --------------------------------- |
| API 능력·요청/응답 필드 형태          | 웹 프레임워크, ORM, 패키지 매니저 |
| 공유 enum (카테고리)                  | 스케줄러/큐 구현                  |
| 데이터 계약 (article 필드, 요약 JSON) | 캐시 구현                         |
| 사전 생성 원칙                        | AI 제공자·모델, 폴더 구조, 배포   |

## 1. 컴포넌트 경계 (개념)

```
[수집기] → [분류·요약기] → [스토리지] → [API] → [프론트]
                                 ↑                ↘ (핫 데이터 캐시)
                         (외부 뉴스 검색) ← [마스코트 검색]
```

- **수집기**: 뉴스 소스 수집, 중복 필터(URL + 본문 유사도 임계값)
- **분류·요약기**: 카테고리 분류 + 요약 3종 생성 (§4 원칙)
- **스토리지**: 기사/요약/사용자 데이터 영속화
- **API**: §3 계약 제공
- **프론트**: 렌더 + 마스코트 인터랙션
- **캐시**: 핫 데이터·검색 결과 (구현·TTL 팀 자유)

각 컴포넌트의 _구현 기술은 정의하지 않습니다._

## 2. 공유 enum — 카테고리

| 내부 값 (저장·API 정본) | 표시 라벨 |
| ----------------------- | --------- |
| `sky`                   | 하늘      |
| `land`                  | 땅        |
| `sea`                   | 바다      |

저장·API는 `sky/land/sea`를 정본으로 사용. 한글은 표시 전용.

## 3. API 계약

> **형태만** 합의. 전송 방식(경로·포트·문서 도구)은 구현 자유지만 아래 **능력과 필드**는 따릅니다.

### 3.1 헬스체크

- 입력 없음 / 출력: `{ "status": "ok" }`

### 3.2 기사 리스트

- 입력: `category`(`sky|land|sea`), `subcategory?`, 페이지네이션(`limit`, `cursor`)
- 출력: 카드 배열 — 필드: `id, title, oneLineSummary, source, sourceLang, publishedAt, thumbnailUrl, category, subcategory`
- `sourceLang`: `ko|en` (원문 언어). 요약·표시 텍스트는 **항상 한국어**

### 3.3 기사 상세

- 입력: `id`
- 출력: 기사 메타 + `sourceLang`(`ko|en`) + `originalUrl` + **원문 전문** + 문단별 요약(§5)
- 원문 전문 저장·응답은 팀플·미배포 한정 — `../hddb_guidelines.md §5.1` 【팀 결정 2026-05-15】
- 영어 소스: 원문은 원어 그대로, 한줄/카드/문단별 요약은 한국어

### 3.4 카테고리 트리

- 출력: 카테고리 + 서브카테고리 트리

### 3.5 사용자 관심사

- 관심사는 **클라이언트 로컬 저장** (계정·인증·서버 식별자 없음). 서버 저장 엔드포인트 불필요
- 개인화 반영 방식은 §6 미결 (요청 파라미터 vs 클라이언트 재정렬)

### 3.6 마스코트 검색

- 입력: `{ "query": "..." }` — 자연어를 **단일 키워드로 축약**해 검색 (멀티턴 없음, stateless)
- 출력: 결과 카드 배열 (§3.2 카드 형태 재사용). 마스코트 톤 텍스트는 프론트 표현 영역

### 에러 형태

- 모든 에러: `{ "error": { "code": <string>, "message": <string> } }` (코드 체계는 합의 후 확장)

## 4. 사전 생성 원칙 (성능 제약 — 합의 필요)

요약(한줄/카드/문단별)은 **클릭 시점이 아니라 수집 시점에 미리 생성**해 저장합니다. 클릭 시 AI 호출은 응답 지연으로 금지. 구현 방식은 자유, **이 원칙만 고정**입니다.

## 5. 데이터 계약

- `article` 최소 필드: §3.2 / §3.3 출력 필드
- 문단별 요약 JSON: `../hddb_guidelines.md §3.1` 구조를 정본 후보로 함 (§6 미해결 항목 참조)

## 6. 미정 / 결정 필요

> **회의 결정 (반영 완료):** 사용자 식별 = 로컬 저장(계정·서버 식별자 없음 — §3.5) · 다국어 = 영어 소스 병행 수집, 요약은 한국어(§3.2/§3.3) · AI = 무료 모델(제공자 팀 자유, 계약 무관) · 마스코트 = 멀티턴 없는 단일 키워드 검색(§3.6).
