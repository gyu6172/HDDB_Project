# 하늘/땅/바다 뉴스 큐레이션 서비스 — 기획

> 자연을 하늘·땅·바다로 분류해 뉴스를 큐레이션하고 AI가 요약을 제공하는 개인화 뉴스 플랫폼.
> 통합 접점(API·enum·데이터 계약)은 [`docs/architecture.md`](./docs/architecture.md) 참고.
> **구현 스택(확정):** 프론트 Next.js · 백엔드 FastAPI + SQLAlchemy · DB Supabase(PostgreSQL + pgvector) · AI 분류 로컬 Ollama, 요약·임베딩 Google Gemini.

## 1. 개요

자연을 **하늘/땅/바다** 세 영역으로 분류해 관련 뉴스를 큐레이션하고 AI 요약을 제공. 중앙 마스코트가 인터랙티브 검색 인터페이스 역할.

차별점:

- "자연/환경" 도메인 특화
- 시각적 메타포(하늘=위, 땅=좌하단, 바다=우하단)로 직관적 탐색
- AI 요약을 마스코트 캐릭터로 의인화

## 2. 주요 기능

- **온보딩**: 하늘/땅/바다별 서브카테고리 다중 선택 → 추천 가중치로 활용
  - 하늘: 새·우주·기상·대기오염 / 땅: 자연재해·동식물·토양오염·곤충 / 바다: 해양생물·심해·해양오염
- **메인 화면**: 상단=하늘, 좌하단=땅, 우하단=바다, 중앙=마스코트+검색. 각 영역에 톱 1~3 기사 한줄 요약, 클릭 시 카테고리 페이지 전환
- **카테고리 페이지**: 카드 그리드(썸네일·제목·한줄요약·출처·발행일). 관심 서브카테고리 상단 배치. 무한 스크롤/페이지네이션
- **기사 상세**: 좌측 원문, 우측 마스코트가 문단별 요약을 말풍선으로 표시. 스크롤 위치와 동기화
- **마스코트 검색**: 자연어 입력 → 질의 임베딩 생성 → 사전 저장된 기사 키워드 임베딩과 의미 유사도(pgvector 코사인) 비교 → 결과 카드 (멀티턴 없는 stateless 검색)

## 3. 데이터 모델

**서버 영속화 (기사 + 분류 마스터):**

- `articles`: 제목, 원문(또는 링크), 출처, 원문 언어(`ko|en`), 발행일, 카테고리, 서브카테고리, 한줄요약, 문단별요약(JSON), 썸네일, 분류 confidence
- `categories` / `subcategories`: 분류 마스터

**클라이언트 로컬 저장 (계정·서버 식별자 없음 — §7 결정):**

- 관심사: 선택 서브카테고리 + weight
- 검색 히스토리: query, timestamp

> 사용자 식별을 로컬 저장으로 확정 → 서버측 `users`/`user_interests`/`search_history` 테이블 불필요.

### 3.1 문단별 요약 JSON 구조

```json
[
  {
    "paragraph_index": 0,
    "original_text": "원문 문단...",
    "summary": "요약된 내용"
  }
]
```

## 4. AI 요약 전략

| 타입        | 용도    | 길이           | 생성 시점      |
| ----------- | ------- | -------------- | -------------- |
| 한줄 요약   | 메인    | 30~50자        | 수집 직후 배치 |
| 카드 요약   | 카드 뷰 | 100자 내외     | 수집 직후 배치 |
| 문단별 요약 | 상세    | 문단당 30~50자 | 수집 직후 배치 |

- **사전 생성 필수**: 클릭 시 AI 호출은 응답 지연으로 금지. 수집 시점에 미리 생성·저장
- **마스코트 톤**: 친근한 자연 가이드 톤("~예요", "~한답니다")을 시스템 프롬프트로 고정
- **언어**: 영어 소스 기사도 한줄/카드/문단별 요약은 **한국어로 생성** (원문은 원어 보존)
- **비용**: 무료 모델 사용(회의 결정). AI 제공자·모델은 팀 자유, 무료 한도·레이트리밋 고려

## 5. 크롤링 전략

- **소스**: 환경/과학 강한 매체 우선. 국내(연합뉴스·한겨레 환경·사이언스타임즈·그린포스트) + **영어 소스 병행**(BBC Earth·Nat Geo·Nature — 회의 결정). RSS 우선
- **분류**: 1차 키워드 룰 → 애매하면 2차 AI 모델 분류. confidence score 함께 저장
- **스케줄링**: 매체별 30분~2시간. 중복 필터: URL + 제목 유사도(코사인 0.85+)

### 5.1 ⚠️ 저작권

원문 전체 저장·노출은 저작권 이슈가 큼. 외부 배포 시 권장: 제목+요약+썸네일+원문 링크, 원문은 외부 링크 처리.

> **【팀 결정 2026-05-15】** 본 프로젝트는 팀플·미배포 전제로 원문 전문 저장·응답을 허용. 외부 배포 검토 시 위 권장 방식 재적용.

## 7. 로드맵 & 결정

**로드맵**: P1 MVP(크롤링 + 한 카테고리 + 기본 메인 + AI 요약) → P2 코어(3 카테고리·온보딩·카드/상세·기본 검색) → P3 개인화(추천·마스코트 고도화·문단 동기화·검색 히스토리) → P4 폴리싱(성능·반응형·애니메이션·에러).

**결정됨 (회의)**: 사용자 식별 = 로컬 저장(계정 없음) · 다국어 = 영어 소스 병행, 요약은 한국어 · AI = 분류 로컬 Ollama, 요약·임베딩 Gemini(무료 한도) · 마스코트 = 멀티턴 없는 임베딩 의미 검색.

**구현 현황**: 통신 계약 확정 및 API 구현 완료(camelCase 응답 · `{items, nextCursor}` 커서 페이지네이션 · 문단별 요약 §3.1 구조 채택). 상세는 [`docs/architecture.md`](./docs/architecture.md) 및 Swagger(`/docs`) 참조.

**리스크**: 저작권 범위 · AI 환각(요약 검증 로직) · 크롤링 차단(robots.txt·요청 빈도) · 무료 모델 한도·레이트리밋 · 검색 히스토리(로컬) 개인정보.
